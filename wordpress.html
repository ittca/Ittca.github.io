Resenha wordpress

Estrutura e comandos essenciais

Para um tema wordpress existem dois ficheiros essenciais:

index.php
style.css

O index pode conter apenas uma estrutura html e o style a decoração que pretendermos, no entanto vários são outros os ficheiros que vão permitir que a plataforma wordpress possa introduzir e mudar dados ao nosso tema, e o wordpress também têm as suas próprias funções para estas questões. As Linguagens permitidas são html, css, javascript e php.
O Style.css deverá começar com os comentários:
/*
Theme Name: resenha wordpress
Author: Tiago Calado
Version: 1.0
*/
vair dar ao tema os dados mencionados, e o wordpress vai lê-los e disponibilizá-los no lugar apropriado.

Esta estrutura minima será o suficiente para que o wordpress assuma os ficheiros como tema, para tal deverá-se criar uma pasta nomeada com o nome do tema.
Dentro da pasta guardam-se estes dois ficheiros e guarda-se a pasta dentro do nosso wordpress ~/wp-content/themes/ .
Neste momento já podemos ir dentro do dashboard do wordpress e nos temas instalados temos o nosso tema que podemos ativar, não vai ter nada, mas está um tema criado e aceite pelo wordpress.

Para adicionar uma foto ao tema, é apenas necessário ter uma foto com umas certas medidas que wordpress aceita, com o nome screenshot.png ou .jpg ou alguns poucos mais formatos em ~/wp-content/themes/ . Desta forma já teremos uma imagem para o nosso tema.

Podemos escrever codigo html no index.php, mas deixariamos de utilizar toda a potencialidade do wordpress, ora para começar a usar as usas funções e dinamismo, pois depois podemos mudar isto tudo dentro do dasboard, podemos começar por escrever no nosso index.php:

<?php
get_header();
get_footer();

Apenas é necessário iniciar o php, não necessitamos de o fechar e com as funções get_header e get_footer o wordpress automaticamente escreve no nosso site os dados que possui, que podem alterados no dasboard.

Ao adicionar mos os ficheiro header.php e footer.php o wp deixa de usar a sua predefinição e vai apresentar no header e no footer o que escrevermos nestes ficheiros.
É aqui que no header.php que vamos abrir a nossa estrutura html até o abrir do corpo (inclusive), a tag fechar body e fechar html vão para o footer.php

header.php
footer.php

Ao usar estes dois ficheiros vamos iniciar a estrutura html no header.php e terminá-la no footer.php, depois no index.php usa-se código php e chama-se no ínicio a get_header() e no fim get_footer().

Adiciona-se dentro do head wp_head().

Adiciona-se antes do fechar da tag body wp_footer().

A partir daqui começa um entrelaçado de colocar aqui e colocar ali e tudo funciona dentro do universo wp na qual é de alguma forma um pouco difícel de escrever fazendo sentido para todos os leitores, isto caso até aqui esteja a fazer sentido.
Mas a estrutura é muito simples e bastante eficaz. Neste momento vou apenas despejar pequenas partes de código que podem ser implementadas, mais tarde talvez explique por escrito ou video.

Adicionar ficheiros .css e .js

function tiagos_pedidos() {
wp_enqueue_style('tiagoscss', get_template_directory_uri().'/css/tiagos.css', array(), '0.0.1', 'all');
wp_enqueue_script('tiagosjs', get_template_directory_uri().'/js/tiagos.js', array(), '0.0.1', true);
}
add_action('wp_enqueue_scripts', 'tiagos_pedidos');

Adicionar menus

function ad_menu(){
add_theme_support('menus'); /* ativa a opção menus no wordpress */
register_nav_menu('menu-primario', 'Menu Principal');
register_nav_menu('menu-secundario', 'Menu Secundario');
}
add_action('init','ad_menu');

Para os menus aparecerem usa-se por exemplo no inicio da tag body
wp_nav_menu(array('theme_location'=>'menu-primario'));

Adicionar opções do wp no personalizar do wp

wp_nav_menu(array('theme_location'=>'menu-primario'));

<img src="<?php header_image(); ?>" height="<?php echo get_custom_header()->height; ?>" width="<?php echo get_custom_header()->width; ?>" alt="">

add_theme_support('post-thumbnails');
the_post_thumbnail('thumbnail');

add_theme_support('post-formats', array('aside', 'image', 'video'));
echo 'This is the format:'.get_post_format()
get_template_part('content', get_post_format()); // vai usar o content.php
O segundo parametro vai fazer procurar se a pagina tem um post_format, se tiver vai usar o ficheiro content-post_format_name.php em vez do content.

Usar páginas especificas

Pode-se criar um template criando um ficheiro de nome page-algo.php e inserindo no inicio o nome do template:
/*
Template Name: Pagina sem titulo
*/
Agora sempre que se cria uma página ou numa já existente pode-se escolher este template.

Pode-se também usar páginas especificas para determinadas páginas usando a slag criando o ficheiro page-slug_name.php ou melhor ainda usando o id especifico da pagina como por exemplo page-24.php

Falta a pasta temporária / sem memória suficiente

info.php com phpinfo();
Dentro do php.ini para a falta da pasta temporária, dar o caminho para uma pasta com esse efeito:
upload_tmp_dir = /tmp
em wp-config.php adicionar mesmo antes de "E é tudo. Pare de editar!" adicionar:
define('WP_TEMP_DIR', dirname(_FILE_). '/wp-content/temp/');
Em wp-content criar uma pasta de nome temp
em linux: sudo chmod a+wrx .../wp-content/temp

Para corrigir a memória de upload permitida alterar em php.ini:
upload_max_filesize = 512M
post_max_size = 512M
